# 运营数据初始化优化计划

## 优化目标

基于对当前运营数据初始化逻辑的深入分析，优化数据结构、初始化流程和持久化机制，确保数据清晰、合理、高效。

---

## 发现的问题

### 问题 1：默认数据可能重复创建
**问题描述：**
- `initDefaultData()` 函数在每次加载时都会检查 `gachaPools.value.length === 0`
- 如果用户手动删除了所有卡池和活动，下次加载时会再次创建默认数据
- 可能导致用户意外数据丢失

**影响：** 用户体验差，数据管理混乱

---

### 问题 2：项目运营数据初始化时机不合理
**问题描述：**
- 项目运营数据只在第一次 `tick()` 时才生成
- 项目发布后到第一次 tick 之间存在数据空窗期
- 可能导致 UI 显示异常或数据访问错误

**影响：** 数据一致性问题，潜在的 bug

---

### 问题 3：recentWelfareValue 重置时机不当
**问题描述：**
- `resetRecentWelfare()` 在保存数据之后调用
- 导致保存到 localStorage 的值总是前一天的
- 数据不一致

**影响：** 福利数据持久化错误

---

### 问题 4：数据清理逻辑分散
**问题描述：**
- 评论和告白的清理逻辑在 `saveToLocal()` 中
- 没有统一的过期数据清理机制
- 可能导致 localStorage 膨胀

**影响：** 存储空间浪费，性能下降

---

### 问题 5：初始化标志位缺失
**问题描述：**
- 没有记录是否已经初始化过默认数据
- 无法区分"首次加载"和"数据被清空"
- 初始化逻辑不够智能

**影响：** 初始化行为不可控

---

## 优化方案

### 优化 1：添加初始化标志位
**实施步骤：**

1. 在 `operationStore.ts` 中添加标志位：
```typescript
const hasInitializedDefaults = ref(false);
```

2. 修改 `initDefaultData()` 函数：
```typescript
function initDefaultData(): void {
  // 如果已经初始化过，直接返回
  if (hasInitializedDefaults.value) {
    return;
  }
  
  // 只在真正首次加载时创建默认数据
  if (gachaPools.value.length === 0 && events.value.length === 0) {
    createDefaultPoolAndEvents();
    hasInitializedDefaults.value = true;
  }
  
  saveToLocal();
}
```

3. 在 `loadFromLocal()` 中加载标志位：
```typescript
function loadFromLocal(): void {
  const saved = localStorage.getItem('operation_data');
  if (saved) {
    const data = JSON.parse(saved);
    // 加载标志位
    if (data.hasInitializedDefaults !== undefined) {
      hasInitializedDefaults.value = data.hasInitializedDefaults;
    }
    // ... 其他加载逻辑
  }
}
```

4. 在 `saveToLocal()` 中保存标志位：
```typescript
function saveToLocal(): void {
  const data = {
    gachaPools: gachaPools.value,
    events: events.value,
    incidents: incidents.value,
    hasInitializedDefaults: hasInitializedDefaults.value,
    // ... 其他数据
  };
  localStorage.setItem('operation_data', JSON.stringify(data));
}
```

### 优化 4：统一数据清理机制
**实施步骤：**

1. 创建独立的数据清理函数（在 `simulationStore.ts` 中）：
```typescript
function cleanupExpiredData(): void {
  const now = Date.now();
  const thirtyDaysAgo = now - (30 * 24 * 60 * 60 * 1000);
  const sevenDaysAgo = now - (7 * 24 * 60 * 60 * 1000);
  
  // 清理过期的评论（超过 30 天）
  projectComments.value.forEach((comments, projectId) => {
    const validComments = comments.filter(c => {
      const commentTime = new Date(c.createdAt).getTime();
      return commentTime > thirtyDaysAgo;
    });
    
    // 只保留最近 50 条
    const limitedComments = validComments.slice(-50);
    
    if (limitedComments.length !== comments.length) {
      projectComments.value.set(projectId, limitedComments);
      console.log(`[Simulation] 清理项目 ${projectId} 的过期评论：${comments.length} -> ${limitedComments.length}`);
    }
  });
  
  // 清理过期的告白（超过 30 天）
  projectConfessions.value.forEach((confessions, projectId) => {
    const validConfessions = confessions.filter(c => {
      const confessionTime = new Date(c.createdAt).getTime();
      return confessionTime > thirtyDaysAgo;
    });
    
    // 只保留最近 20 条
    const limitedConfessions = validConfessions.slice(-20);
    
    if (limitedConfessions.length !== confessions.length) {
      projectConfessions.value.set(projectId, limitedConfessions);
      console.log(`[Simulation] 清理项目 ${projectId} 的过期告白：${confessions.length} -> ${limitedConfessions.length}`);
    }
  });
  
  // 清理过期项目运营数据（项目已删除或超过 7 天未更新）
  projectOperationData.value.forEach((data, projectId) => {
    const project = projectStore.projects.find(p => p.id === projectId);
    if (!project) {
      // 项目已删除，清除数据
      projectOperationData.value.delete(projectId);
      console.log(`[Simulation] 清理已删除项目 ${projectId} 的运营数据`);
    }
  });
}
```

2. 在 `tick()` 函数中调用清理函数：
```typescript
async function tick(): Promise<SimulationTickResult | null> {
  // ... 前面的逻辑
  
  // 10. 清理过期数据
  console.log('[Simulation] 清理过期数据...');
  cleanupExpiredData();
  
  // 11. 重置每日福利值
  operationStore.resetRecentWelfare();
  
  // 12. 保存数据
  saveToLocal();
  
  // ... 后续逻辑
}
```

3. 从 `saveToLocal()` 中移除清理逻辑，保持职责单一

### 优化 5：优化数据结构
**实施步骤：**

1. 在 `operationStore.ts` 中添加数据类型定义：
```typescript
export interface OperationStoreData {
  gachaPools: GachaPool[];
  events: GameEvent[];
  incidents: OperationIncident[];
  hasInitializedDefaults: boolean;
  newOperationPredictions: Array<[string, NewOperationPrediction]>;
  activeOperationEffects: Array<[string, ActiveOperationEffect]>;
  completedOperationEffects: PredictionVsActual[];
  recentWelfareValue: number;
  welfareHistory: Array<...>;
}
```

2. 使用类型安全的保存函数：
```typescript
function saveToLocal(): void {
  const data: OperationStoreData = {
    gachaPools: gachaPools.value,
    events: events.value,
    incidents: incidents.value,
    hasInitializedDefaults: hasInitializedDefaults.value,
    newOperationPredictions: Array.from(newOperationPredictions.value.entries()),
    activeOperationEffects: Array.from(activeOperationEffects.value.entries()),
    completedOperationEffects: completedOperationEffects.value,
    recentWelfareValue: recentWelfareValue.value,
    welfareHistory: welfareHistory.value
  };
  
  try {
    localStorage.setItem('operation_data', JSON.stringify(data));
    console.log('[Operation] 数据保存成功');
  } catch (error) {
    console.error('[Operation] 数据保存失败:', error);
    throw error;
  }
}
```

## 风险评估

### 风险 1：数据兼容性问题
**描述：** 修改数据结构可能导致旧存档无法加载

**缓解措施：**
- 在 `loadFromLocal()` 中添加数据迁移逻辑
- 检查数据版本，自动升级到新格式
- 提供数据备份功能

### 风险 2：初始化逻辑改变影响现有功能
**描述：** 修改初始化时机可能影响依赖默认数据的功能

**缓解措施：**
- 全面测试所有依赖默认数据的功能
- 保留向后兼容性
- 逐步迁移，先测试后上线

### 风险 3：数据清理误删重要数据
**描述：** 清理逻辑可能误删用户想要保留的数据

**缓解措施：**
- 清理前备份数据
- 提供清理日志
- 添加配置选项允许用户自定义清理策略
