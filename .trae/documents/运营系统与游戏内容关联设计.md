# 运营系统与游戏内容关联设计计划

## 一、现状问题分析

### 当前核心问题
1. **数据完全脱节**：simulationStore 使用虚拟数据，与 projectStore/gameStore 无关联
2. **模拟无意义**："模拟一天"只是随机数生成，不反映玩家决策效果
3. **运营数据虚假**：收入、玩家数、评论等都与真实项目无关
4. **反馈不真实**：告白/同人使用硬编码角色，而非玩家创建的角色

### 当前数据流向
```
玩家创建项目 → 发布游戏 → ??? （断点）
                          ↓
              simulationStore.tick() 使用虚拟数据
                          ↓
              显示"假"运营数据
```

## 二、设计目标

### 核心目标
建立**真实的运营反馈闭环**：
```
玩家决策（角色/剧情/活动/卡池）→ 影响模拟计算 → 产生真实数据 → 指导下一步决策
```

### 设计原则
1. **真实性**：所有运营数据必须基于玩家创建的内容
2. **可感知**：玩家能清楚看到决策如何影响运营结果
3. **策略性**：不同决策产生不同结果，有优劣之分
4. **成长性**：随着项目质量提升，运营数据应持续改善

### 3.2 核心关联矩阵

| 玩家决策 | 影响指标 | 计算方式 | 数据验证 |
|---------|---------|---------|---------|
| **角色质量** | 抽卡收入 | 人气值 × 爆率合理性 × 玩家数 | 可对比不同角色UP收益 |
| **角色人气** | 玩家留存 | 高人气角色 → 高留存意愿 | 可追踪人气与留存相关性 |
| **剧情质量** | 玩家满意度 | 剧情评分 × 完成率 | 可对比不同剧情满意度 |
| **剧情标签** | 评论内容 | 剧情标签匹配 → 正面评论 | 可验证评论与剧情关联 |
| **爆率设置** | 玩家满意度 | 实际爆率 vs 期望爆率 | 可对比不同爆率满意度 |
| **活动配置** | 参与度/收入 | 活动类型 × 奖励价值 × 难度 | 可对比不同活动效果 |
| **福利发放** | 短期留存 | 福利价值 × 发放时机 | 可验证福利ROI |

## 四、详细设计方案

### 4.1 项目质量评分系统

#### 4.1.1 角色综合质量评分
```typescript
interface CharacterQualityScore {
  // 基础属性 (40%)
  baseScore: number;           // 外貌/性格/背景等基础设定质量
  
  // 人气指标 (30%)
  popularityScore: number;     // 当前人气值 / 100
  
  // 互动深度 (20%)
  interactionScore: number;    // 亲密度系统完整度 + 约会场景数
  
  // 美术质量 (10%)
  artScore: number;            // AI生成质量评分
  
  // 综合评分
  totalScore: number;          // 加权平均
}

// 计算方式
function calculateCharacterQuality(character: Character): number {
  const baseScore = character.attributes ? 0.7 : 0.5;
  const popularityScore = character.popularity?.base || 50 / 100;
  const interactionScore = (character.intimacy?.length || 0) / 10;
  const artScore = character.visual?.quality || 0.7;
  
  return baseScore * 0.4 + popularityScore * 0.3 + interactionScore * 0.2 + artScore * 0.1;
}
```

#### 4.1.2 剧情综合质量评分
```typescript
interface PlotQualityScore {
  // 结构完整度 (30%)
  structureScore: number;      // 章节数 / 预期章节数
  
  // 分支丰富度 (25%)
  branchScore: number;         // 分支节点数 / 预期节点数
  
  // 内容质量 (25%)
  contentScore: number;        // AI评分 / 100
  
  // 标签匹配度 (20%)
  tagMatchScore: number;       // 剧情标签与角色标签匹配度
  
  totalScore: number;
}

// 计算方式
function calculatePlotQuality(plot: Plot): number {
  const structureScore = Math.min(1, plot.chapters?.length / 10);
  const branchScore = Math.min(1, plot.nodes?.length / 20);
  const contentScore = (plot.quality || 70) / 100;
  const tagMatchScore = calculateTagMatch(plot.tags, plot.characters);
  
  return structureScore * 0.3 + branchScore * 0.25 + contentScore * 0.25 + tagMatchScore * 0.2;
}
```

### 4.2 运营模拟引擎重构

#### 4.2.1 核心计算公式

**1. 玩家满意度计算**
```typescript
function calculateSatisfaction(
  plotQuality: number,        // 剧情质量 (0-1)
  rateFairness: number,       // 爆率合理性 (0-1)
  welfareSatisfaction: number // 福利满意度 (0-1)
): number {
  // 基础满意度
  let satisfaction = 0.5;
  
  // 剧情质量影响 (40%)
  satisfaction += (plotQuality - 0.5) * 0.4;
  
  // 爆率公平性影响 (35%)
  satisfaction += (rateFairness - 0.5) * 0.35;
  
  // 福利满意度影响 (25%)
  satisfaction += (welfareSatisfaction - 0.5) * 0.25;
  
  // 限制在 0-1 范围
  return Math.max(0, Math.min(1, satisfaction));
}
```

**2. 留存率计算**
```typescript
function calculateRetention(
  satisfaction: number,       // 玩家满意度
  characterAppeal: number,    // 角色吸引力 (平均人气)
  activityRichness: number,   // 活动丰富度
  communityVibe: number       // 社区氛围 (评论情感)
): number {
  // 基础留存率 60%
  let retention = 0.6;
  
  // 满意度影响 (40%)
  retention += (satisfaction - 0.5) * 0.4;
  
  // 角色吸引力影响 (30%)
  retention += (characterAppeal - 0.5) * 0.3;
  
  // 活动丰富度影响 (20%)
  retention += (activityRichness - 0.5) * 0.2;
  
  // 社区氛围影响 (10%)
  retention += (communityVibe - 0.5) * 0.1;
  
  return Math.max(0.1, Math.min(0.95, retention));
}
```

**3. 付费率计算**
```typescript
function calculatePayRate(
  characterPopularity: number, // 角色平均人气
  gachaAppeal: number,        // 卡池吸引力 (UP角色人气)
  economyBalance: number      // 经济系统平衡度
): number {
  // 基础付费率 5%
  let payRate = 0.05;
  
  // 角色人气影响 (40%)
  payRate += characterPopularity * 0.15;
  
  // 卡池吸引力影响 (40%)
  payRate += gachaAppeal * 0.15;
  
  // 经济平衡影响 (20%)
  payRate += (economyBalance - 0.5) * 0.05;
  
  return Math.max(0.01, Math.min(0.5, payRate));
}
```

**4. 收入计算**
```typescript
function calculateRevenue(
  activePlayers: number,      // 活跃玩家数
  payRate: number,            // 付费率
  avgSpending: number,        // 人均付费 (基于卡池配置)
  characterBonus: number      // 角色人气加成
): number {
  const payingPlayers = activePlayers * payRate;
  const baseRevenue = payingPlayers * avgSpending;
  
  // 应用角色人气加成
  return Math.floor(baseRevenue * characterBonus);
}
```

**5. 评论情感计算**
```typescript
function calculateCommentSentiment(
  satisfaction: number,       // 玩家满意度
  plotTagMatch: number,       // 剧情标签匹配度
  recentEvents: string[]      // 近期运营事件
): 'positive' | 'neutral' | 'negative' {
  let sentimentScore = satisfaction;
  
  // 剧情标签匹配加分
  sentimentScore += (plotTagMatch - 0.5) * 0.2;
  
  // 运营事件影响
  recentEvents.forEach(event => {
    if (event === 'good_welfare') sentimentScore += 0.1;
    if (event === 'rate_controversy') sentimentScore -= 0.2;
    if (event === 'plot_backlash') sentimentScore -= 0.15;
  });
  
  if (sentimentScore > 0.6) return 'positive';
  if (sentimentScore < 0.4) return 'negative';
  return 'neutral';
}
```

### 4.3 真实数据生成

#### 4.3.1 基于真实角色的评论生成
```typescript
function generateRealCharacterComments(
  characters: Character[],
  sentiment: 'positive' | 'neutral' | 'negative'
): Comment[] {
  const comments: Comment[] = [];
  
  characters.forEach(char => {
    // 根据角色属性选择模板
    const templates = getTemplatesForCharacter(char, sentiment);
    
    // 生成 1-3 条评论
    const count = 1 + Math.floor(Math.random() * 3);
    for (let i = 0; i < count; i++) {
      const template = templates[Math.floor(Math.random() * templates.length)];
      comments.push({
        content: template.replace('{name}', char.name),
        sentiment,
        characterId: char.id,
        platform: getRandomPlatform()
      });
    }
  });
  
  return comments;
}
```

function generateRealConfessions(
  characters: Character[],
  playerSatisfaction: number
): Confession[] {
  const confessions: Confession[] = [];
  
  // 根据角色人气和玩家满意度决定告白数量
  characters.forEach(char => {
    const probability = (char.popularity?.base || 50) / 100 * playerSatisfaction;
    
    if (Math.random() < probability) {
      confessions.push({
        characterId: char.id,
        characterName: char.name,
        content: generateConfessionContent(char),
        likes: Math.floor(Math.random() * 100 * (char.popularity?.base || 50) / 50)
      });
    }
  });
  
  return confessions;
}

## 七、风险评估

### 技术风险
1. **计算复杂度增加** → 需要优化算法和添加缓存
2. **数据一致性维护** → 需要完善同步机制
3. **历史数据兼容** → 需要设计迁移方案

### 设计风险
1. **公式平衡性** → 需要大量测试调整参数
2. **玩家预期管理** → 需要清晰的反馈说明
