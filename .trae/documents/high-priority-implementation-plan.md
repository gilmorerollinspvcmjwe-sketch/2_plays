# 高优先级内容实现计划

## 目标
实现之前分析报告中确定的高优先级任务，完善核心功能。

## 高优先级任务清单

### 1. 评论系统完善 (gacha-player-system SubTask 2.2)
**目标**: 扩展 commentStore.ts，添加平台维度字段和互动功能

**具体步骤**:
1. 修改 `src/stores/commentStore.ts`
   - 添加 `platform` 字段到 GameComment 接口
   - 添加 `isLiked`, `isShared`, `isReplied` 等互动状态字段
   - 实现 `likeComment()`, `shareComment()`, `replyComment()` 方法
   
2. 修改 `src/components/community/PlatformComments.vue`
   - 添加点赞/分享/回复按钮
   - 实现互动状态显示（实心/空心图标）
   - 添加互动动画效果

3. 修改 `src/data/templates/platformComments/index.ts`
   - 为每个平台添加特有 emoji 和梗
   - 添加平台风格配置（颜色、字体等）

**预计时间**: 2 小时

---

### 2. GameDetailView 角色人气展示 (gacha-player-system SubTask 6.2)
**目标**: 在游戏详情页显示角色人气值、讨论热度、CP 热度

**具体步骤**:
1. 修改 `src/views/game/GameDetailView.vue`
   - 添加角色人气卡片组件
   - 显示人气值（0-100 进度条）
   - 显示讨论热度（数字 + 趋势箭头）
   - 显示 CP 热度（排行榜形式）
   
2. 添加人气趋势图
   - 使用图表库（如 Chart.js 或 ECharts）
   - 显示近 7 天人气变化
   - 添加时间筛选器（7天/30天/90天）

3. 添加样式
   - 人气值颜色区分（高/中/低）
   - 卡片悬停效果
   - 响应式布局

**预计时间**: 3 小时

---

### 3. 事件模板补充 (otome-creator-sim SubTask 2.5)
**目标**: 创建 20 个事件模板，完善运营事件系统

**具体步骤**:
1. 创建 `src/data/templates/incidents/` 目录结构
   - 按类型分类：dropRate（爆率）、plot（剧情）、welfare（福利）、other（其他）
   
2. 创建爆率争议事件（5 个）
   - 百连无 SSR
   - UP 角色歪卡
   - 保底机制质疑
   - 概率虚假宣传
   - 非酋抱团取暖

3. 创建剧情雷点事件（5 个）
   - 角色 OOC（Out of Character）
   - CP 党争
   - 剧情逻辑漏洞
   - 结局烂尾
   - 角色死亡

4. 创建福利节奏事件（5 个）
   - 对比竞品福利
   - 要求增加签到奖励
   - 节日福利不足
   - 补偿不到位
   - 氪金体验差

5. 创建其他事件（5 个）
   - 服务器宕机
   - 数据丢失
   - 运营事故
   - 抄袭争议
   - 公关危机

**预计时间**: 2 小时
