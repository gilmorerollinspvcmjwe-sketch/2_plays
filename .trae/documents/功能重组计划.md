# 功能重组计划

## 目标
重新组织游戏功能入口，优化用户体验：
1. 将"团队管理"和"创建项目"移到【游戏开发】页面
2. 将"角色排行"和"剧情分析"移到【运营】页面
3. 将"角色排行"与运营页面的"卡池管理"结合

---

## 当前功能分布

### 主页 (HomeView.vue)
- 公司数据看板（创建项目按钮、团队管理按钮）
- 游戏开发快速入口（创建角色、设计剧情、剧情分析、角色排行）
- 待处理事项
- 我的项目
- 市场动态

### 游戏开发 (CreatorEntryView.vue)
- 创建角色
- 设计剧情
- 剧情分析
- 角色排行

### 运营 (OperationView.vue)
- 卡池管理（含角色人气排行）
- 活动中心
- 运营事件
- 玩家数据
- 福利发放
- 告白墙
- 同人广场
- 市场情报
- 效果追踪

---

## 重组方案

### 方案一：完全重组（推荐）

#### 主页简化
```
主页：
├── 公司数据看板（仅显示资金，无操作按钮）
├── 游戏开发入口（点击进入开发页面）
├── 运营入口（点击进入运营页面）
├── 待处理事项
├── 我的项目
└── 市场动态
```

#### 游戏开发页面重组
```
游戏开发页面：
├── 当前项目信息
├── 快速操作区
│   ├── [创建项目] 按钮
│   ├── [团队管理] 按钮
│   ├── [创建角色] 按钮
│   └── [设计剧情] 按钮
├── 开发进度
└── 项目列表
```

#### 运营页面重组
```
运营页面：
├── 运营数据概览
├── Tab切换
│   ├── 角色分析（原角色排行 + 卡池管理）
│   │   ├── 角色人气排行
│   │   ├── 角色价值分析
│   │   └── 卡池管理
│   ├── 剧情分析（新增Tab）
│   │   ├── 剧情完成率
│   │   ├── 流失点分析
│   │   └── 剧情热度排行
│   ├── 活动中心
│   ├── 运营事件
│   ├── 玩家数据
│   ├── 福利发放
│   ├── 告白墙
│   ├── 同人广场
│   ├── 市场情报
│   └── 效果追踪
```

### Phase 7: 测试验证

- [ ] 验证数据流正确

## 预期效果

1. **主页更简洁** - 只保留核心信息和两个主要入口
2. **功能分组更合理** - 开发相关功能集中在游戏开发页，运营相关功能集中在运营页
3. **角色分析更完整** - 人气排行、价值分析、卡池管理三位一体
4. **剧情分析独立** - 给剧情分析独立空间，便于后续扩展
