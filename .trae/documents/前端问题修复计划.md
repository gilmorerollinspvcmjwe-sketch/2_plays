### 阶段三：添加路由守卫

#### 步骤 3.1: 为公司创建页面添加路由守卫
**文件**: `src/router/index.ts`

**修改内容**:
```typescript
// 在路由配置中添加 meta
{
  path: 'company-setup',
  name: 'companySetup',
  component: CompanySetupView,
  meta: { 
    requiresNoCompany: true,
    title: '创建公司'
  }
}

// 在路由守卫中添加检查
router.beforeEach((to, from, next) => {
  const companyStore = useCompanyStore();
  
  // 如果路由需要无公司状态但用户已有公司
  if (to.meta.requiresNoCompany && companyStore.hasCompany) {
    next('/home');  // 重定向到首页
  } else {
    next();
  }
});
```

**验证方法**:
1. 创建公司后访问 `/company-setup`
2. 应该自动重定向到 `/home`
3. 未创建公司时可以正常访问

### 阶段四：优化 CreatorProfile 数据流

#### 步骤 4.1: 统一技能升级逻辑
**文件**: `src/views/ProfileView.vue`

**修改内容**:
```typescript
async function handleUpgradeSkill(type: SkillType) {
  // 1. 检查技能点
  const skillPoints = creatorStore.availableSkillPoints;
  if (skillPoints <= 0) {
    showToast('技能点不足，升级创作者等级可获得技能点');
    return;
  }
  
  // 2. 检查积分
  if (pointsStore.balance < 100) {
    showToast('积分不足，需要 100 积分');
    return;
  }
  
  // 3. 先扣积分
  const pointsResult = await pointsStore.spendPoints(100, '技能升级');
  if (!pointsResult.success) {
    showToast('积分扣除失败，请重试');
    return;
  }
  
  // 4. 再升级技能
  const upgradeResult = creatorStore.upgradeSkill(type);
  if (!upgradeResult.success) {
    // 5. 失败返还积分
    await pointsStore.spendPoints(-100, '技能升级失败返还');
    showToast(upgradeResult.message);
    return;
  }
  
  // 6. 成功提示
  showToast(`${upgradeResult.message}，消耗 100 积分`);
}
```

**说明**: 这个函数已经实现得很好，只需确保逻辑顺序正确
