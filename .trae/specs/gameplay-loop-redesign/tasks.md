# Tasks - 乙游模拟器核心玩法重构

## 系统架构概述

本系统采用**双层模拟架构**：

```
┌─────────────────────────────────────────────────────────┐
│                    玩家公司模拟层                         │
│  (玩家作为游戏公司老板/策划，管理项目、团队、资源)        │
└─────────────────────┬───────────────────────────────────┘
                      │ "下一天" 触发
                      ▼
┌─────────────────────────────────────────────────────────┐
│               玩家模拟系统 (核心模拟引擎)                 │
│  ┌──────────┐ ┌──────────┐ ┌──────────┐ ┌──────────┐ │
│  │ 运营模拟 │ │ 评论模拟 │ │ 玩家行为  │ │ 市场模拟 │ │
│  │  子系统  │ │  子系统  │ │  子系统  │ │  子系统  │ │
│  └──────────┘ └──────────┘ └──────────┘ └──────────┘ │
│                    ↑                                     │
│         (规则驱动 + 随机性 + Tag + AI)                  │
└─────────────────────┬───────────────────────────────────┘
                      │
                      ▼
┌─────────────────────────────────────────────────────────┐
│                世界变化系统 (环境层)                      │
│  - 其他游戏公司状态                                      │
│  - 市场热点/趋势变化                                     │
│  - 行业事件                                             │
│  - 赛季制题材热度轮换                                   │
└─────────────────────────────────────────────────────────┘
```

**核心交互**：玩家点击"下一天" → 触发玩家模拟系统 → 系统计算所有子系统的真实变化 → 更新公司/项目/角色/剧情数据

## 优先级说明

- **P0 (Critical)**：核心功能，必须优先完成，其他功能依赖这些。开发难度：中等
- **P1 (High)**：重要功能，对玩家体验有显著影响。开发难度：中等偏高
- **P2 (Medium)**：增强功能，提升游戏深度。开发难度：中等
- **P3 (Low)**：优化功能，可以后续迭代。开发难度：低

优先级评估维度：
- **依赖关系**：被其他功能依赖的优先
- **开发难度**：简单功能优先，复杂功能后置
- **玩家体验**：核心体验功能优先

4. **M4前评估**：是否需要调整数值平衡？

# 优先级评估说明

## P0 (Critical) - 为什么这些是必须的？

1. **Task 1 (项目数据模型)**：
   - 依赖：无，所有其他功能都依赖它
   - 难度：中等
   - 原因：没有项目概念，整个游戏就没有核心

2. **Task 2 (员工系统)**：
   - 依赖：Task 1（员工要分配到项目）
   - 难度：中等
   - 原因：流程要求"组建团队"在创建角色之前

3. **Task 3 (首页重构)**：
   - 依赖：Task 1, Task 2
   - 难度：中等
   - 原因：首页是玩家主要交互界面，必须展示项目

4. **Task 4 (项目评估)**：
   - 依赖：Task 1
   - 难度：中等偏高
   - 原因：让玩家了解项目状态，是后续改进的基础

## P1 (High) - 为什么这些是重要的？

5. **Task 5 (运营联动)**：
   - 依赖：Task 1
   - 难度：高（最复杂的功能）
   - 原因：实现"真实模拟"的核心，但难度高所以放在P1

6. **Task 6 (角色价值)** 和 **Task 7 (剧情分析)**：
   - 依赖：Task 4
   - 难度：中等
   - 原因：4大计算系统的一部分，复用Task 4的引擎

7. **Task 8 (数据闭环)**：
   - 依赖：Task 5, 6, 7
   - 难度：高
   - 原因：让玩家知道如何改进游戏，需要前面所有计算系统

## P2 (Medium) - 增强体验

8. **Task 9 (运营预测)**：
   - 依赖：Task 5
   - 难度：高
   - 原因：锦上添花，让玩家看到预测，但不是必须的

9. **Task 10 (赛季制)** 和 **Task 11 (竞争环境)**：
   - 依赖：Task 5, 10
   - 难度：中等
   - 原因：长期玩法机制，前期可以没有

## P3 (Low) - 优化迭代

10. **Task 12-15**：
    - 依赖：前面的任务
    - 难度：低-中等
    - 原因：完善功能，可以后续迭代
