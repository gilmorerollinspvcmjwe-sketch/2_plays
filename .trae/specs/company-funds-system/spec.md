# 公司资金系统设计规范

## Why
当前游戏缺少统一的公司资金系统，项目收入、员工工资、运营成本等没有关联到公司层面的资金流动。需要建立一个简化的资金系统，让收入有去处、支出有来源，增强游戏的真实感和策略性。

## What Changes
- **新增 companyFunds 到 companyStore**: 统一管理公司资金
- **收入入口(3个)**: 项目运营收入、投资事件、成就奖励
- **支出口(4个)**: 员工工资、项目开发成本、运营成本、福利发放
- **与现有系统集成**: 在 simulationStore.tick() 中每日结算资金
- **UI更新**: 主页显示实时资金，资金不足时给出提示和限制

## Impact
- Affected files:
  - `src/stores/companyStore.ts` - 添加 funds 字段和相关方法
  - `src/stores/simulationStore.ts` - 每日结算时更新资金
  - `src/views/HomeView.vue` - 显示实时资金
  - `src/components/project/CompanyDashboard.vue` - 资金展示

## ADDED Requirements

### Requirement: 公司资金模型
公司资金 SHALL 在 companyStore 中统一管理，包含初始资金、收入、支出逻辑。

#### Scenario: 资金初始化
- **WHEN** 创建新公司时
- **THEN** 初始资金为 50万
- **AND** 资金持久化存储到 localStorage

#### Scenario: 资金收入
- **WHEN** 每日模拟结算时
- **THEN** 所有运营项目的 dailyRevenue 汇总存入公司资金
- **AND** 显示收入明细（项目A收入X元，项目B收入Y元）

#### Scenario: 资金支出-员工工资
- **WHEN** 每日模拟结算时
- **THEN** 扣除所有员工当日工资总和
- **AND** 工资 = 员工年薪 / 365（简化按日计算）
- **AND** 资金不足时员工满意度下降

#### Scenario: 资金支出-运营成本
- **WHEN** 每日模拟结算时
- **THEN** 扣除基础运营成本（固定5000元/天）
- **AND** 每个运营项目额外增加2000元/天运营成本

#### Scenario: 资金不足处理
- **WHEN** 资金不足以支付工资时
- **THEN** 员工满意度每日下降5%
- **AND** 触发"资金紧张"事件
- **AND** 无法创建新项目、无法招聘员工

### Requirement: 收入数值设计
基于乙游行业简化模型设计收入数值。

#### Scenario: 项目收入计算
```
日收入 = DAU × 付费率 × ARPU
其中：
- DAU: 根据项目质量和运营活动计算（1000-50000）
- 付费率: 5%-15%（行业平均8%）
- ARPU: 5-50元（根据卡池和活动浮动）

示例：
- 小型项目：1000 DAU × 5% × 10元 = 500元/天
- 中型项目：10000 DAU × 8% × 20元 = 16000元/天
- 大型项目：50000 DAU × 12% × 30元 = 180000元/天
```

#### Scenario: 特殊收入
- **投资事件**: 随机触发，一次性收入 10万-100万
- **成就奖励**: 完成成就奖励 1万-10万

### Requirement: 支出数值设计
基于乙游公司运营成本简化模型。

#### Scenario: 员工工资标准
```
职位等级与年薪：
- 初级(Junior): 8-12万/年 → 日薪 220-330元
- 中级(Middle): 15-25万/年 → 日薪 410-685元
- 高级(Senior): 30-50万/年 → 日薪 820-1370元
- 专家(Expert): 60-100万/年 → 日薪 1640-2740元

示例团队日工资（5人）：
- 1高级策划 + 1中级美术 + 1中级程序 + 2初级运营
- = 1000 + 550 + 550 + 250 + 250 = 2600元/天
```

#### Scenario: 运营成本
```
基础运营成本: 5000元/天（房租、水电、服务器等）
项目运营成本: 2000元/天/项目（推广、客服、活动）

示例：
- 无项目时: 5000元/天
- 2个项目运营: 5000 + 4000 = 9000元/天
```

#### Scenario: 开发成本
```
创建新项目时一次性扣除：
- 立项费用: 10万元
- 开发期间每日投入: 5000元（已包含在基础运营中）
```

### Requirement: 与现有系统集成
资金系统 SHALL 与现有的 simulationStore、employeeStore、projectStore 无缝集成。

#### Scenario: 每日结算流程
```
1. simulationStore.tick() 开始
2. 计算所有运营项目收入 → 累加到 companyFunds
3. 计算员工工资总额 → 从 companyFunds 扣除
4. 计算运营成本 → 从 companyFunds 扣除
5. 检查资金状态（充足/紧张/枯竭）
6. 触发相关事件
7. 保存资金数据
```

#### Scenario: 创建项目检查
- **WHEN** 用户点击"创建项目"
- **THEN** 检查 companyFunds >= 10万
- **AND** 不足时提示"资金不足，需要10万元立项费用"

#### Scenario: 招聘员工检查
- **WHEN** 用户招聘员工时
- **THEN** 检查当前资金能否支付该员工1个月工资
- **AND** 不足时提示"资金不足以支付新员工工资"

## 数值平衡表

| 项目 | 数值 | 说明 |
|------|------|------|
| 初始资金 | 50万 | 游戏开始资金 |
| 小型项目日收入 | 500-2000 | 刚上线项目 |
| 中型项目日收入 | 5000-20000 | 稳定运营项目 |
| 大型项目日收入 | 50000-200000 | 爆款项目 |
| 员工日工资 | 200-3000 | 根据等级 |
| 基础运营成本 | 5000/天 | 固定支出 |
| 项目运营成本 | 2000/天/项目 | 可变支出 |
| 立项费用 | 10万 | 一次性 |

## 资金状态定义

| 状态 | 条件 | 影响 |
|------|------|------|
| 充足 | > 30万 | 正常运营 |
| 紧张 | 10万-30万 | 提示注意，不能创建新项目 |
| 枯竭 | < 10万 | 不能招聘，员工满意度下降 |
| 破产 | < 0 | 游戏结束或强制融资 |
