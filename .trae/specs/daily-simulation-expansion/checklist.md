# Checklist

## Phase 1: 基础架构验证

- [ ] 员工和角色类型扩展正确集成到现有系统
- [ ] 每日事件引擎能正确生成三类事件（正面、负面、中性）
- [ ] 角色每日更新引擎能正确计算各项数据

---

## Phase 2: 员工系统验证

- [ ] 疲劳度动态管理正确运行：
  - [ ] 工作中员工疲劳度正确增长（+10/天，性格有差异）
  - [ ] 休息员工疲劳度正确恢复（-30/天）
  - [ ] 项目完成后疲劳度正确恢复（-20）
- [ ] 满意度与项目表现挂钩正确：
  - [ ] 项目成功时满意度 +3
  - [ ] 项目连续失败时满意度 -5
- [ ] 员工随机事件触发概率正确：
  - [ ] 生病事件（1%/天）
  - [ ] 获奖事件（5%/天）
  - [ ] 请假事件（疲劳>80 且满意度<40 时 10%）
- [ ] 经验增长逻辑正确（基于项目进度）
- [ ] 团队默契度系统正确：
  - [ ] 默契度每日增长（+1/天，7 天后开始）
  - [ ] 默契度加成正确应用（5-15%）

---

## Phase 3: 角色系统验证

- [ ] 亲密度衰减正确（每日 0.5%）
- [ ] 人气变化基于项目表现正确：
  - [ ] 收入>预期 20%: +5
  - [ ] 收入>预期：+2
  - [ ] 收入=预期：不变
  - [ ] 收入<预期：-2
  - [ ] 收入<预期 50%: -5
- [ ] 生日系统正确：
  - [ ] 正确检测生日日期
  - [ ] 触发时人气 +20、热度 +50
  - [ ] 自动生成生日祝福评论（5-10 条）
- [ ] 评论提及统计正确：
  - [ ] 正确识别评论中的角色
  - [ ] 正面评论人气 +1，负面 -1
- [ ] CP 热度计算正确
- [ ] 口碑评分计算正确（10 条以上评论时）

---

- [ ] 季节加成正确：

- [ ] 节假日效应正确：

  - [ ] 情人节：恋爱题材收入 +50%、人气 +10、告白 +200%
  - [ ] 项目周年庆：收入 +200%、人气 +20
- [ ] 特殊日期 buff 正确：
  - [ ] 周末：活跃 +30%、收入 +20%
  - [ ] 寒暑假：学生向项目活跃 +50%、收入 +40%
