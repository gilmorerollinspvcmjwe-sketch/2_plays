# 乙女游戏创作者模拟器 Spec

## Why
为乙女游戏爱好者提供一款轻松有趣的**模拟经营游戏**，让玩家从消费者转变为创作者，体验乙女游戏开发与运营的乐趣。游戏强调**低门槛、高反馈、强反差感**，通过模板化、智能化的设计工具，让玩家无需专业开发知识即可享受创作乐趣，核心爽点在于看到玩家反馈、掌控爆率、打造爆款乙游的成就感。

## What Changes
- 创建**模拟器**而非专业编辑器，强调易上手、可玩性强
- 提供丰富的**预设模板**（角色模板、剧情模板、活动模板），一键套用
- 简化配置操作，采用**选择题**而非填空题的设计方式
- 核心体验聚焦在**玩家反馈的反差感**（从被骂到被夸、从非酋到欧皇）
- 实现 AI 驱动的玩家反馈系统，增强真实感与沉浸感
- **BREAKING**: 首次创建，无破坏性变更

## Impact
- 影响范围：全新项目，无现有代码影响
- 核心系统：游戏引擎、UI 系统、数据管理系统、反馈模拟系统
- 目标平台：PC/移动端（后续扩展）

## ADDED Requirements

### Requirement: 游戏开发系统（简化版）
系统 SHALL 提供**模板化**的乙女游戏开发工具，强调**易上手**：
- **角色创建模板**：提供预设角色类型（霸道总裁、温柔学长、腹黑少爷等），玩家只需选择并微调
  * 预设人设模板（10+ 经典乙游人设）
  * 一键换装系统（服装、发型、配饰模板）
  * 性格标签选择（傲娇、温柔、高冷等，而非复杂参数）
- **剧情模板系统**：提供经典剧情套路模板
  * 甜宠路线模板（相遇→暧昧→告白→交往）
  * 虐恋路线模板（相遇→误会→分离→重逢）
  * 悬疑路线模板（初遇→线索→真相→结局）
  * 玩家只需填充关键节点，AI 自动补全过渡剧情
- **互动配置**：简化互动设置
  * 预设互动包（触摸反应、语音触发、约会场景）
  * 好感度系统一键启用（无需配置复杂数值）
- **美术风格选择**：提供 3-5 种主流画风模板（日系、韩系、国风），一键切换
- **声优阵容**：提供预设声优包（新人、资深、顶流），按预算选择

#### Scenario: 快速创建角色
- **WHEN** 玩家创建新角色
- **THEN** 从预设模板中选择"霸道总裁"类型，微调外貌和性格标签，30 秒内完成创建

### Requirement: 运营管理系统（轻量化）
系统 SHALL 提供**轻量化**的运营功能，降低操作复杂度：
- **活动模板库**：提供经典活动模板，一键套用
  * 节日活动模板（情人节、七夕、圣诞节）
  * 角色生日活动模板（限定剧情、专属卡池）
  * 联动活动模板（IP 联动、跨游戏联动）
- **卡池设计简化**：提供预设爆率方案
  * 欧皇方案（SSR 爆率 5%）
  * 普通方案（SSR 爆率 2%）
  * 非酋方案（SSR 爆率 0.5%）
  * 玩家可滑动调整爆率，实时看到预期收益变化
- **一键运营**：提供自动化运营选项
  * 自动活动策划（AI 根据时间节点推荐活动）
  * 自动爆率调整（根据玩家反馈智能调整）
  * 自动危机公关（负面事件自动处理）
- **数据面板**：简化数据展示，突出关键指标
  * 收入趋势图（日/周/月）
  * 玩家满意度评分（1-5 星）
  * 热门角色/剧情排行

#### Scenario: 快速策划活动
- **WHEN** 玩家需要策划情人节活动
- **THEN** 选择"情人节活动模板"，微调活动奖励和卡池，1 分钟内完成策划

### Requirement: 市场反馈系统（核心爽点）
系统 SHALL 打造**真实且有趣**的玩家反馈体验，突出**反差感**：
- **AI 驱动的玩家评论**：模拟真实乙女游戏玩家社区
  * 评论区模拟（TapTap、微博、B 站风格）
  * 玩家吐槽生成（"卡池太非了"、"剧情刀傻了"、"老公好帅"）
  * 安利向评论（"姐妹们快冲"、"这个卡面绝美"）
  * 节奏带师（带节奏、引战、对比其他游戏）
- **反馈情感极端化**：放大玩家情绪，增强反差感
  * 好评时：疯狂安利、"神仙游戏"、"老公我 can"
  * 差评时：疯狂吐槽、"退钱"、"再也不氪了"
  * 从被骂到被夸的成就感爆棚
- **玩家群体分化**：模拟不同玩家群体
  * 氪金大佬（在意爆率、强度）
  * 剧情党（在意剧情质量、角色塑造）
  * 外观党（在意立绘、CG 质量）
  * 休闲玩家（在意活动福利、上手难度）
- **舆情事件系统**：制造戏剧性事件
  * 爆率争议事件（玩家集体非酋，要求提高爆率）
  * 剧情雷点事件（某角色党不满剧情走向）
  * CP 党争事件（不同 CP 粉丝互撕）
  * 福利节奏事件（对比竞品福利，要求加码）

#### Scenario: 玩家反馈反差感
- **WHEN** 玩家调整爆率从 0.5% 提升到 2%
- **THEN** 评论区从"退钱垃圾"瞬间变成"良心官方"、"老婆我爱你"，形成强烈反差

### Requirement: 资源管理系统（简化经济）
系统 SHALL 提供**易于理解**的经济模型，避免复杂计算：
- **简化资源类型**：只保留核心资源
  * 金币（通用货币）
  * 钻石（氪金货币）
  * 人气值（玩家满意度）
  * 开发点数（用于开发新功能）
- **直观的成本展示**：用时间而非金额展示成本
  * "这个活动需要开发 3 天"
  * "这个卡池需要投入 10 万金币"
  * 避免复杂的 ROI 计算
- **一键平衡**：提供预设的资源分配方案
  * 稳健型（平衡开发与营销）
  * 激进型（重注营销，快速扩张）
  * 保守型（注重开发，稳步发展）
- **风险提示**：简单明了的风险提示
  * "这样可能导致资金链断裂"
  * "玩家可能会因为福利少而流失"

#### Scenario: 资源分配
- **WHEN** 玩家需要决定新版本预算
- **THEN** 选择"稳健型"方案，系统自动分配资源到开发、营销、福利

### Requirement: AI 内容生成系统（增强反差感）
系统 SHALL 利用 AI 技术**增强真实感与反差感**，而非增加复杂度：
- **AI 玩家评论生成**：自动生成真实感爆棚的玩家评论
  * 使用乙女游戏圈用语（"老公"、"老婆"、"非酋"、"非天"）
  * 生成具体细节（"这个卡面眼神绝了"、"剧情里那个摸头杀我哭了"）
  * 情绪化表达（"啊啊啊"、"！！！",增强感染力）
  * 玩梗能力（结合当下热门梗）
- **AI 舆情事件生成**：制造戏剧性事件
  * 根据玩家决策生成合理事件（爆率低→非酋节奏）
  * 事件升级机制（小节奏→大节奏→危机）
  * 事件解决反馈（处理后玩家态度转变）
- **智能难度调节**：根据玩家表现调整难度
  * 新手期：玩家反馈更宽容，增强信心
  * 成长期：玩家要求提高，增加挑战
  * 高难度模式：玩家更挑剔，成就感更强
- **反差感强化**：AI 刻意放大情绪反差
  * 从"垃圾游戏"到"神仙游戏"的评论转变
  * 从"退钱"到"再氪一单"的玩家行为转变
  * 从"凉透了"到"出圈了"的口碑转变

#### Scenario: AI 增强反差感
- **WHEN** 玩家连续几次卡池爆率设置很低后被骂
- **THEN** 玩家提高爆率后，AI 生成大量"良心发现"、"这还是那个铁公鸡吗"等评论，形成强烈反差

### Requirement: 核心爽点设计
系统 SHALL 突出**"从被骂到被夸"**的反差爽感：
- **身份转变**：从"被玩家骂的设计师"到"被玩家夸的神仙官方"
  * 开局可能因为爆率低被骂"铁公鸡"
  * 调整后变成"良心官方"，形成强烈反差
- **掌控感**：掌握"爆率"、"福利"、"剧情走向"的生杀大权
  * 想让玩家"非"就"非"，想让玩家"欧"就"欧"
  * 想发"刀"就发"刀"，想发"糖"就发"糖"
- **即时反馈**：决策后立刻看到玩家反应
  * 调高爆率→评论区"老婆我爱你"
  * 发虐心剧情→评论区"刀片已寄"
  * 发福利→评论区"神仙游戏"
- **成就系统**：记录"逆袭"历程
  * "从被骂到被夸"成就
  * "逆风翻盘"成就
  * "出圈了"成就
- **低门槛高上限**：
  * 新手：用模板就能玩得开心
  * 老手：深度调优追求完美数据

## 技术架构（纯 H5 第一版）

### 前端技术栈
- **核心框架**：Vue3 + TypeScript + Vite
- **UI 组件库**：Vant UI 4.x（移动端专属，轻量级）
- **状态管理**：Pinia（轻量，TypeScript 友好）
- **路由管理**：Vue Router 4.x
- **HTTP 客户端**：Axios（请求拦截、错误处理）
- **样式方案**：TailwindCSS + SCSS（原子化 CSS）
- **PWA 支持**：vite-plugin-pwa（离线缓存、添加到桌面）

### 后端技术栈
- **核心框架**：Node.js + Express（轻量快速）或 Python + FastAPI（AI 集成方便）
- **API 规范**：RESTful API + OpenAPI 文档
- **认证方案**：JWT Token（无状态认证）
- **文件存储**：本地存储（第一版）/ 阿里云 OSS（后续扩展）

### 数据库：Supabase
- **核心优势**：
  * 开源 Firebase 替代，免费额度充足
  * 内置 PostgreSQL，支持复杂查询
  * 实时订阅功能（玩家反馈实时更新）
  * 内置认证系统（用户登录、权限）
  * 自动 API 生成（减少后端开发量）
- **数据表设计**：
  * users（用户表）
  * games（游戏表）
  * characters（角色表）
  * plots（剧情表）
  * events（活动表）
  * comments（评论表）
  * analytics（数据表）

### AI 模型集成（多模型支持）
- **支持的国产大模型**：
  * MiniMax 2.5（ABAB 系列，文本生成强）
  * Kimi 2.5（月之暗面，长文本处理）
  * DeepSeek（深度求索，代码能力强）
  * 豆包（字节跳动，多模态）
  * 通义千问 3.5（阿里，综合能力强）
  * 文心一言 4.0（百度，中文优化）
- **AI 接入方案**：
  * 统一接口层：封装各家 API，统一输入输出格式
  * 智能切换：主模型失败时自动切换备用模型
  * 成本控制：根据功能重要性选择不同模型
- **AI 使用场景**：
  * 玩家评论生成（主要用 MiniMax、Kimi）
  * 舆情事件生成（主要用通义千问、豆包）
  * 剧情补全（主要用 Kimi 长文本）
  * 角色设定生成（主要用 DeepSeek）
  * **AI 图片生成（收费项目）**：接入即梦 2.0，生成角色立绘、CG

### 预设素材库（兜底方案）
- **必要性**：应对 AI 模型连接失败、成本高、响应慢
- **预设模板数量**：至少 100 个
- **模板分类**：
  * 角色模板（30 个）：霸道总裁、温柔学长等经典人设
  * 剧情模板（20 个）：甜宠、虐恋、悬疑等路线
  * 评论模板（50 个）：吐槽、安利、节奏等评论类型
  * 活动模板（20 个）：节日、生日、联动等活动
  * 事件模板（20 个）：爆率争议、剧情雷点等事件
- **兜底逻辑**：
  * AI 请求超时（>10 秒）→ 使用预设模板
  * AI 请求失败 → 使用预设模板
  * 用户选择"快速模式" → 使用预设模板
  * 用户积分不足（AI 图片生成） → 使用预设素材

### 轻图片重文字策略
- **核心理念**：用文字描述代替复杂 UI，降低开发成本，提升可玩性
- **文字化设计**：
  * 角色创建：文字描述外貌（"剑眉星目"、"温柔笑容"）而非调整五官 UI
  * 服装设计：文字选择（"白色衬衫"、"黑色西装"）而非换装 UI
  * 场景描述：文字生成（"樱花飘落的公园"）而非精美背景图
  * 互动设计：文字选项（"摸头"、"拥抱"、"告白"）而非复杂动画
- **图片使用原则**：
  * 仅关键位置使用图片（角色立绘、重要 CG）
  * 使用预制素材库（免费/付费素材）
  * AI 图片生成作为收费项目（即梦 2.0）
  * 默认使用占位图 + 文字描述
- **成本优势**：
  * 文字 LLM 成本：0.01 元/次（低）
  * 图片生成成本：0.1-1 元/次（高）
  * 预设素材：一次性成本

## 移动端 UI 设计原则（轻图片重文字）

### 文字化交互设计
- **角色创建**：
  * 外貌描述：选择题形式（"剑眉星目"、"温润如玉"、"冷峻棱角"）

  * 背景故事：预设模板选择 + AI 润色
- **剧情设计**：

  * 剧情走向：分支选项（"上前搭话"、"默默观察"、"转身离开"）
  * 对话内容：预设对话模板 + AI 生成
- **活动设计**：
  * 活动类型：选择器（"情人节限定"、"角色生日"、"节日庆典"）
  * 活动奖励：文字选择（"限定卡池"、"专属剧情"、"头像框"）

### 图片使用策略
- **必要图片**：
  * 角色立绘：预制素材库（默认） / AI 生成（收费）
  * 重要 CG：预制素材库（默认） / AI 生成（收费）
  * UI 背景：简单渐变色 + 文字
- **占位方案**：
  * 默认头像：灰色剪影 + 角色名
  * 默认 CG：纯色背景 + 场景描述文字
  * 图标：使用 emoji 或简单 SVG

- **配色方案**：粉色系为主（#FFB6C1、#FFC0CB、#FF69B4）
- **字体选择**：圆体/手写体（增强少女感）
- **文案风格**：乙女游戏用语（"老公"、"老婆"、"非酋"、"欧皇"）

- **选择器**：底部弹窗选择器（文字选项）

- **竖屏优先**：单手握持，核心功能在下半屏
- **卡片布局**：信息分块，易于滑动浏览
- **底部导航**：3-5 个主入口（首页、开发、运营、评论、我的）
