# Vue 3 项目白屏问题诊断与解决（完整版）

## 问题概述

用户报告 Vue 3 + Vite + Pinia 项目加载后显示白屏且页面无响应（卡死）。经过多轮诊断和修复，最终定位并解决了问题。

## 问题诊断过程

### 第一阶段：Store 循环依赖诊断与修复

**现象**：
- 页面加载后完全白屏
- 页面无响应，控制台可能有错误

**初步分析**：
- Pinia store 之间的循环依赖
- stores 在 `defineStore` 顶层直接调用 `useXxxStore()` 导致初始化死循环

**发现的具体问题**：
1. **14 处 store 之间直接调用**：
   - `points.ts` 调用 `useGameStore()`
   - `gameStore.ts` 调用 `usePointsStore()`
   - `commentStore.ts` 调用多个 store
   - `operationStore.ts` 调用多个 store

2. **循环依赖示例**：
```typescript
// ❌ 错误示例
export const usePointsStore = defineStore('points', () => {
  const gameStore = useGameStore()  // 触发 gameStore 初始化
  // ...
})

export const useGameStore = defineStore('game', () => {
  const pointsStore = usePointsStore()  // 触发 pointsStore 初始化
  // ...
})
```

**尝试的解决方案**：
1. ✅ **创建 EventBus 事件驱动架构** - `src/utils/eventBus.ts`
2. ✅ **修改所有 stores 使用事件通信**：
   - `points.ts` - 注释掉 14 个 `eventBus.on()` 监听器
   - `gameStore.ts` - 注释掉事件监听器
   - `commentStore.ts` - 改为发射 `GAME_UPDATE_CHARACTER_FROM_COMMENT` 事件
   - `operationStore.ts` - 改为发射 `POINTS_SPEND` 和 `COMMENT_GENERATE` 事件
3. 移除 vite.config.ts 中的 auto-import 插件
4. 移除所有 `eventBus.on()` 监听器防止无限循环

**结果**：❌ 仍然白屏（说明核心问题不在 EventBus，而在初始化时机）

### 第二阶段：深度分析

**分析范围**：
- 检查所有 store 之间的依赖关系（14 处直接调用）
- 检查 App.vue、HomeView.vue 的初始化逻辑
- 检查模板文件和路由配置

**发现的问题**：
1. `MainLayout.vue` 在 setup 阶段直接调用 `usePointsStore()`
2. `HomeView.vue` 在 onMounted 中动态导入多个 store
3. `gameStore.initDefaultGame()` 和 `TemplateManager.getCharactersByPopularity()` 可能导致性能问题

**尝试的解决方案**：
1. 简化 App.vue 为最小测试页面
2. 移除 Pinia，只使用纯 Vue
3. 修改 vite.config.ts 移除所有插件

**结果**：❌ 仍然白屏

### 第五阶段：代码问题修复

#### 问题 1：main.ts 缺少路由注册

**原始代码**：
```typescript

import App from './App.vue'

```

**修复后**：
```typescript

import App from './App.vue'
import router from './router'  // ✅ 添加路由导入

const app = createApp(App)
const pinia = createPinia()
app.use(pinia)
app.use(router)  // ✅ 注册路由
app.mount('#app')
```

### 最佳实践

#### 1. Store 初始化

- ✅ **避免在 defineStore 顶层调用其他 store**
  ```typescript
  // ❌ 错误
  export const useStoreA = defineStore('a', () => {
    const storeB = useStoreB()  // 循环依赖
  })
  
  // ✅ 正确 - 使用 EventBus 解耦
  eventBus.emit('EVENT_NAME', data)
  ```

- ✅ **避免在组件 setup 顶层调用 store**
  ```typescript
  // ❌ 错误
  const pointsStore = usePointsStore()
  
  // ✅ 正确 - 延迟到 onMounted
  onMounted(async () => {
    const module = await import('@/stores/points')
    pointsStore.value = module.usePointsStore()
  })
  ```

- ✅ **使用动态导入延迟初始化**
  ```typescript
  let store: any = null
  onMounted(async () => {
    const module = await import('@/stores/xxx')
    store = module.useXxxStore()
  })
  ```

#### 2. Store 之间通信

- ✅ **使用 EventBus 解耦**
  ```typescript
  // 发射事件
  eventBus.emit('POINTS_SPEND', { amount: 100 })
  
  // 监听事件（在 onMounted 中注册）
  onMounted(() => {
    eventBus.on('POINTS_SPEND', handleSpend)
  })
  ```

- ✅ **避免直接 import 其他 store**
  ```typescript
  // ❌ 错误
  import { useOtherStore } from './otherStore'
  
  // ✅ 正确
  eventBus.emit('EVENT', data)
  ```

## 结论

本次白屏问题由**四个独立问题**叠加导致：

### 1. 浏览器缓存/扩展干扰（主要问题）
- 浏览器缓存或扩展插件干扰了页面的正常加载
- **解决方案**：使用隐私模式或清除缓存

### 2. 代码中路由未注册（关键问题）
- main.ts 中缺少 `app.use(router)` 导致 `<router-view />` 无法解析
- **解决方案**：在 main.ts 中添加路由注册

### 3. Store 循环依赖 + 初始化过早（次要问题）
- 14 处 store 之间直接调用导致循环依赖
- MainLayout.vue 和 HomeView.vue 在 setup 阶段调用 store
- **解决方案**：
  - 使用 EventBus 解耦 store 之间的通信
  - 将 store 初始化延迟到 onMounted
  - 使用动态导入确保 Pinia 已初始化

### 4. 组件变量未定义（细节问题）
- PlatformComments.vue 使用了未定义的 `currentPlatform` 变量
- **解决方案**：改为使用已定义的 `activePlatform` 变量

通过系统化的诊断方法，从浏览器到代码逐层排查，使用控制变量法定位问题组件，最终定位并解决了所有问题。
